## 12 段式落地页结构

```
┌─────────────────────────────────────────────┐
│ 1. HERO              │ 首屏可见,3秒定生死    │
│                       │ 主标题 + 副标题 + CTA │
├───────────────────────┤───────────────────────┤
│ 2. SOCIAL PROOF       │ 客户Logo墙/数字       │
├───────────────────────┤───────────────────────┤
│ 3. PROBLEM            │ 痛点场景(用户共鸣)    │
├───────────────────────┤───────────────────────┤
│ 4. SOLUTION           │ 解决方案概述           │
├───────────────────────┤───────────────────────┤
│ 5. FEATURES           │ Bento Grid 功能展示    │
├───────────────────────┤───────────────────────┤
│ 6. HOW IT WORKS       │ 3-5 步骤流程图         │
├───────────────────────┤───────────────────────┤
│ 7. DEMO / VIDEO       │ 产品演示截图/视频      │
├───────────────────────┤───────────────────────┤
│ 8. TESTIMONIALS       │ 用户评价轮播           │
├───────────────────────┤───────────────────────┤
│ 9. PRICING            │ 价格对比表             │
├───────────────────────┤───────────────────────┤
│ 10. FAQ               │ 折叠式问答             │
├───────────────────────┤───────────────────────┤
│ 11. FINAL CTA         │ 最终行动号召           │
├───────────────────────┤───────────────────────┤
│ 12. FOOTER            │ 链接 + Sitemap         │
└───────────────────────┴───────────────────────┘
```

## 各段位设计要点

### 1. Hero

| 元素 | 规范 |
|------|------|
| 主标题 | ≤12字, 说清产品核心价值 |
| 副标题 | ≤25字, 补充说明或社会证明 |
| CTA 按钮 | 1个主CTA, 动词语气, 高对比色 |
| 视觉 | 产品截图 / 动态演示 / 抽象插画 |
| 高度 | 100vh (min-height: 100dvh) |

**CTA 文案公式** (来源: humanizer-zh):
- ✅ "免费开始使用" / "查看演示" / "下载试用"
- ❌ "开启您的旅程" / "探索无限可能" / "立即体验革命性的..."

### 2. Social Proof

| 类型 | 适用场景 |
|------|----------|
| Logo 墙 | B2B SaaS (知名客户) |
| 大数字 | B2C (用户量/下载量/好评率) |
| 评价摘录 | 所有类型 |
| 媒体报道 | 融资/获奖 |

### 3. Problem & 4. Solution

```
Problem: 用用户的语言描述痛点
  "你是否也遇到过...[具体场景]?"

Solution: 用产品功能回答痛点
  "[产品名] 通过 [核心机制] 解决这个问题"
```

### 5. Features (Bento Grid)

```html
<!-- 推荐布局: Bento Grid 3-6个功能卡片 -->
<div class="bento-grid">
  <div class="feature-card large">  <!-- 主功能,占2列 -->
    <icon> <h3>功能名</h3> <p>具体能做什么,数据支撑</p>
  </div>
  <div class="feature-card">        <!-- 辅功能,占1列 -->
    ...
  </div>
</div>
```

**文案规则**: 每个功能 = 功能名 + 1句话 + 1个具体数字/对比

### 8. Testimonials

| 元素 | 规范 |
|------|------|
| 头像+姓名+职位 | 增加可信度 |
| 引用内容 | 具体说解决了什么问题 |
| 避免 | "产品很好用!"这类空洞评价 |
| 展示方式 | 轮播 / 网格 / 瀑布流 |

### 9. Pricing

```
推荐3档: Free / Pro / Enterprise
├── 突出推荐档(Pro) — 加边框/标签"最受欢迎"
├── Free 档: 有限但可用,引导升级
├── Pro 档: 核心价值,性价比最高
└── Enterprise: 定制化,联系销售
```

## 落地页风格适配

| 产品类型 | Hero 视觉 | 配色 | CTA 风格 |
|----------|-----------|------|----------|
| AI SaaS | 动态粒子/渐变 | 暗色+霓虹强调 | "开始免费试用" |
| 电商 | 高品质产品图 | 暖色+品牌色 | "立即购买" |
| 金融 | 数据图表/安全视觉 | 深蓝+金色 | "开通账户" |
| 健康 | 自然场景/人物 | 绿色+奶油白 | "预约咨询" |
| 游戏 | 游戏画面/角色 | 霓虹+暗色 | "立即下载" |
| 教育 | 学习场景/书籍 | 蓝色+白色 | "开始学习" |

## 落地页性能优化

- Hero 图片: WebP, ≤200KB, LCP < 2.5s
- CTA 按钮: 首屏立即可点击(无 JS 阻塞)
- 懒加载: 第 3 段起的所有图片
- 字体: subset 化, preload 仅 Display 字体
- 第三方脚本: 异步加载(GTM/分析/聊天插件)

## CTA 文案速查 (来源: humanizer-zh)

按产品类型匹配,避免 AI 痕迹:

| 产品类型 | ✅ 好的 CTA | ❌ AI味的 CTA |
|----------|-----------|-------------|
| SaaS 工具 | 免费开始 / 创建项目 | 开启您的数字化转型之旅 |
| 电商 | 加入购物车 / 立即购买 | 探索我们的精选系列 |
| 内容/媒体 | 订阅 / 免费阅读 | 解锁独家优质内容 |
| 社区 | 加入 / 注册账号 | 成为我们蓬勃发展的社区一员 |
| 金融 | 开通账户 / 开始投资 | 为您的财务未来赋能 |
| 教育 | 开始学习 / 报名 | 释放您的全部潜能 |
