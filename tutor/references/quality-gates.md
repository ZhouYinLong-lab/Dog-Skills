# 质量门禁与验收规则

## 章节完成门禁（12 项）

每章完成前必须**同时满足**以下所有条件，缺一不可：

1. ✅ 已生成教程 Markdown：`output/tutorials/chXX.md`
2. ✅ 已生成习题课 Markdown：`output/exercise_classes/chXX.md`
3. ✅ 已生成覆盖审计文件：`output/audit/chXX_coverage.csv`
4. ✅ 已生成 Markdown 自检文件：`output/audit/chXX_md_check.txt`
5. ✅ 已生成文本质量自检文件：`output/audit/chXX_text_quality_check.txt`
6. ✅ 若本章使用图片，已生成图片审计文件：`output/audit/chXX_image_audit.csv`
7. ✅ 所有 Markdown 图片链接均有效
8. ✅ 所有图片均为局部裁剪图，不得是整页截图
9. ✅ "本章常见考法"与"典型题型总结"覆盖关系通过检查（≥80%）
10. ✅ 所有必须掌握考点均有证据来源
11. ✅ 所有核心公式均有解释，不得只罗列公式
12. ✅ 所有例题均有详细讲解，不得只给答案

**如果以上任一项未完成，不得输出"本章完成报告"，只能输出"本章未完成问题清单"**（模板见 `references/report-templates.md`）。

## 文本质量验收规则

### 规则 1：考法-题型覆盖

"本章常见考法"中列出的每一种考法，必须在"典型题型总结"中至少有一个对应题型。覆盖率达到 80% 以上才算通过。

### 规则 2：考点证据

每个"必须掌握"考点，必须至少关联以下证据之一：
- 往年卷题号（如"2021 年第 3 题"）
- 课后习题题号
- 教材典型例题
- 用户明确要求

### 规则 3：知识点解释完整性

每个核心知识点必须包含以下全部维度：
- 概念本质
- 为什么引入
- 公式每一项含义
- 做题识别信号
- 使用条件
- 常见误区
- 至少一个例题或反例

### 规则 4：考法展开充分性

如果某章前面列出多类常见考法，后面只展开 1-2 类，判定为不合格。

### 规则 5：题型数量门槛

典型题型总结数量不得少于本章高频考法数量的 80%。

### 规则 6：例题讲解完整性

每个例题必须说明：
- 来源
- 对应知识点
- 第一步为什么这样做
- 中间关键步骤为什么成立
- 答案如何检查
- 常见错误

## 内容剔除规则

### 剔除来源

考试无关内容来自两个方面（取并集）：
1. 用户在 `source/user_scope.md` 或 prompt 中明确指出的不考内容
2. Agent 通过分析往年卷、往年卷答案、课后习题答案后判断出的不考内容

**最终剔除范围 = 用户明确指定不考 ∪ Agent 根据往年卷和习题判断不考**

### 剔除约束

- 不能误删理解核心考点所必需的前置概念
- 如果某内容本身不直接考，但理解高频考点必须依赖它，压缩保留
- 所有删除、压缩、保留决定都必须记录到审计文件
- 不允许无理由删除教材内容

## 高质量章节判定标准

一个章节可以称为"完成"的标准：

1. 教程不是简单公式汇总，而是能解释学生为什么要这样理解
2. 习题课不是答案堆砌，而是能讲清楚题型识别和做题思路
3. 图片不是整页截图，而是围绕知识点的局部清晰截图
4. 审计文件能说明保留、压缩保留、剔除的依据
5. 自检文件能发现并记录 Markdown、公式、图片和文本质量问题
6. 完成报告能真实反映未解决风险，不得为了完成而隐瞒问题
