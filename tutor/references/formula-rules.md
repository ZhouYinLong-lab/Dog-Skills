# 数学公式格式强制规则

生成的 Markdown 文件将使用 KaTeX（VS Code Markdown 预览或类似渲染器）渲染数学公式。必须严格遵守以下规则。

## 基本定界符

- 行内公式：`$...$`
- 独立公式（display math）：`$$...$$`
- **禁止**使用 `\(...\)` 和 `\[...\]`

## 独立公式规则

### 1. `$$` 必须单独占行

**正确**：
```md
令一阶偏导为零。

$$
f_x = -(1+e^y)\sin x
$$

因此只需讨论 $\sin x$ 的符号。
```

**错误**：
```md
$$ f_x = -(1+e^y)\sin x $$
```

### 2. 独立公式前后必须各有一个空行

**正确**：公式前一行是空行，公式后一行是空行。

**错误**：公式紧挨着正文文字。

### 3. 不要连续堆多个独立公式块

如果几个公式属于同一步推导，合并到一个 `aligned` 环境中：

**正确**：
```md
$$
\begin{aligned}
f_x &= -(1+e^y)\sin x \\
f_y &= e^y(\cos x-1-y)
\end{aligned}
$$
```

**错误**：
```md
$$
f_x=-(1+e^y)\sin x
$$

$$
f_y=e^y(\cos x-1-y)
$$
```

## 禁止事项

### 1. 禁止中文进入公式环境

**正确**：
```md
因为 $1+e^y>0$，所以只需令

$$
\sin x=0
$$
```

**错误**：
```md
$$
因为 1+e^y>0，所以 \sin x=0
$$
```

### 2. 禁止在 `$$...$$` 内部再使用 `$`

**正确**：
```md
$$
f(2k\pi,0)=2
$$
```

**错误**：
```md
$$
$f(2k\pi,0)=2$
$$
```

内部出现 `$` 会导致 KaTeX 报错：`KaTeX parse error: Can't use function '$' in math mode`。

### 3. 禁止使用非标准数学函数名

所有数学函数必须使用标准 LaTeX 命令：

| 正确 | 错误 |
|------|------|
| `\sin x` | `sin x` |
| `\cos x` | `cos x` |
| `\ln x` | `ln x` |
| `\lim` | `lim` |
| `\max` | `max` |
| `\min` | `min` |

## 列表中的公式

在有序/无序列表中插入独立公式时，公式必须与当前列表项对齐（缩进 3 空格）：

**正确**：
```md
1. 先计算一阶偏导。

   $$
   \begin{aligned}
   f_x &= -(1+e^y)\sin x \\
   f_y &= e^y(\cos x-1-y)
   \end{aligned}
   $$

   因为 $1+e^y>0$ 且 $e^y>0$，所以后续只需要讨论 $\sin x$ 和 $\cos x-1-y$。

2. 再令一阶偏导为零。
```

**错误**：公式没有缩进，破坏了列表结构。

## 常用 LaTeX 写法速查

| 用途 | 写法 |
|------|------|
| 正弦 | `\sin x` |
| 余弦 | `\cos x` |
| 自然对数 | `\ln x` |
| 指数 | `e^x` |
| 分数 | `\frac{a}{b}` |
| 根号 | `\sqrt{x}` |
| 圆周率 | `\pi` |
| 正无穷 | `+\infty` |
| 趋近于 | `\to` |
| 偏导 | `f_x`, `f_y`, `f_{xx}`, `f_{xy}`, `f_{yy}` |
| 极限 | `\lim_{x\to+\infty}` |
| 积分 | `\int_a^b f(x)\,dx` |
| 求和 | `\sum_{n=1}^\infty` |
| 偏导符号 | `\partial` |
| 梯度 | `\nabla` |
| 点乘 | `\cdot` |
| 对齐环境 | `\begin{aligned} ... \end{aligned}` |
| 矩阵 | `\begin{pmatrix} ... \end{pmatrix}` |

## 写完后的检查步骤

1. 检查所有 `$$` 是否成对出现
2. 检查所有 `$` 是否成对出现（行内公式）
3. 检查所有 `$$` 是否单独占一行
4. 检查独立公式前后是否有空行
5. 检查公式块内部是否有 `$`
6. 检查是否有 `\(` 或 `\[` 残留
7. 在 VS Code 中打开 Markdown 预览，确认无 KaTeX 报错
8. 若有报错，优先排查：`\(...\)` 残留 > `$$` 内嵌 `$` > 中文误入公式 > 花括号不匹配
