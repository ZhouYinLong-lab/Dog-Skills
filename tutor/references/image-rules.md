# 图片处理规则

## 哪些图必须保留

如果教材中出现以下类型的图，且对理解或做题有帮助，必须从 PDF 截取：

- 电路图、电子线路图
- 受力图、光路图、波形图
- 曲线图、函数图像
- 结构图、模型图、示意图
- 表格截图（复杂表格无法用 Markdown 还原时）
- 任何仅靠文字描述难以理解的图

如果是微积分等以公式为主的课程，文字描述已足够、图中无额外信息的情形，可以不生成图片，但必须在自检文件中说明"本章无必要图片"。

## 图片截取规则

### 绝对禁止

**禁止使用整页截图冒充图像。** 每张图片必须只包含目标图/表及其图号图题，不得包含：
- 大段正文
- 页眉、页脚、页码
- 整页留白
- 完整页面标题或段落

### 正确做法

1. 定位目标图像在 PDF 中的页码范围
2. 对每个候选图像记录：图号、图题、所在页码、对应知识点、是否必须保留
3. 对必须保留的图像，用局部裁剪提取——裁剪边界贴近目标图，四周只保留少量边距（建议 10-20px）
4. 允许保留图号和图题（如"图 6.1"）
5. 保存到 `output/assets/chXX/`，清晰命名，如：
   - `figure_03_01_bjt_model.png`
   - `figure_05_02_common_source.png`
   - `table_07_01_integral_formulas.png`

### 使用脚本提取

使用 `scripts/extract_images.py` 从 PDF 指定页码和坐标区域提取图片：

```bash
python scripts/extract_images.py <pdf_path> <page_num> <x0> <y0> <x1> <y1> <output_path>
```

如果无法可靠定位目标图像边界，不得用整页截图替代。必须执行以下之一：
1. 使用更精细的方法重新提取（如 PyMuPDF 的图像定位功能）
2. 将该图片列入 `output/audit/chXX_image_audit.csv`，标记为"需人工裁剪"
3. 在 Markdown 中暂不插入该图，并在完成报告中说明原因

## Markdown 引用格式

```md
![BJT 小信号模型](../assets/ch03/figure_03_01_bjt_model.png)
```

使用相对路径，从 `output/tutorials/` 或 `output/exercise_classes/` 指向 `output/assets/`。

## 图片质量检查

每张保存的图片必须通过以下检查：

1. 图片宽度 ≥ 400 px（除非原图本身很小）
2. 目标图像占图片面积的 50% 以上
3. 无关正文区域不超过 30%
4. 不得包含完整页面标题、整页段落、页码和大面积空白
5. 若图片是整页截图，必须判定为不合格并重新裁剪
6. Markdown 中引用图片前，必须打开图片文件确认视觉效果

## 图片审计

每章使用图片时，必须生成 `output/audit/chXX_image_audit.csv`（格式见 `references/audit-spec.md`）。

如果某张图片未通过检查，`是否通过检查` 必须为"否"，且不得在教程 Markdown 中作为正式图片引用，除非在完成报告中明确说明"该图需人工复核"。

## 处理流程

每个章节的图片处理顺序：

1. 定位教材页中所有候选图像
2. 记录：图号、图题、PDF 页码、对应知识点、是否必须保留
3. 对必须保留的图像进行局部裁剪
4. 保存到 `output/assets/chXX/`
5. 打开保存后的文件检查是否为局部图像（非整页截图）
6. 检查 Markdown 中的图片链接是否有效
7. 在审计文件中记录每张图片的检查结果
