# 期末复习资料生成规则

## 总目标

本项目的目标是：根据教材 PDF、往年卷 PDF、课后习题答案 PDF、往年卷答案 PDF，以及用户给出的考试范围，生成适合期末复习的 Markdown 教程和习题课讲义。

最终产物包括两类：

1. 逐章教程 Markdown
   输出到：`output/tutorials/chXX.md`

2. 逐章习题课 Markdown
   输出到：`output/exercise_classes/chXX.md`

同时，如果教材中有电路图、物理图、曲线图、结构图、模型图、表格截图等需要保留的图像，必须从 PDF 中截取原图，放到：

`output/assets/chXX/`

并在 Markdown 中使用相对路径链接，使预览时可以直接显示。

---

## 输入材料

材料位于：

* `source`：教材、课后习题答案、往年卷及其答案

处理任何章节前，必须先阅读相关源文件，不能只凭常识或记忆生成。

---

## 核心剔除规则

逐章生成教程时，必须剔除书上所有和考试无关的内容。

考试无关内容来自两个来源：

1. 用户在 `source/user_scope.md` 或 prompt 中明确指出的不考内容。
2. Agent 自己通过分析往年卷、往年卷答案、课后习题答案后判断出的不考内容。

最终剔除范围为：

用户明确指定不考的内容 ∪ Agent 根据往年卷和习题判断不考的内容

也就是说，只要属于以上任一来源，就应从教程正文中剔除。

但是，剔除时必须满足以下要求：

* 不能误删理解核心考点所必需的前置概念。
* 如果某内容本身不直接考，但理解高频考点必须依赖它，可以压缩保留。
* 所有删除、压缩、保留决定都必须记录到审计文件中。
* 不允许无理由删除教材内容。

---

## 内容保留规则

对于保留下来的所有有用内容，必须尽量保留教材原文，包括但不限于：

* 定义
* 定理
* 公式
* 符号说明
* 重要结论
* 教材中的核心解释
* 典型例题题干
* 关键推导步骤
* 图表标题和说明

允许对保留下来的内容进行重新排列，使结构更清晰、更适合复习，但不得改变原意。

重排时必须做到：

* 逻辑顺序更清楚；
* 先讲概念，再讲公式，再讲用途，再讲例题；
* 对同一知识点分散在教材不同位置的内容，可以合并到同一节；
* 如果教材某处没有讲清楚，但另一处例题或习题中体现了该思想，应在教程中合并解释。

## 解释生成原则

解释必须以“学生真正看懂”为目标。

每讲一个知识点，都要主动构思读者可能提出的理解问题，并直接回答这些问题。

解释时必须包括：

* 这个概念本质上在说什么；
* 为什么要引入这个概念；
* 它和前后知识有什么关系；
* 公式每一项的物理意义或数学意义；
* 公式为什么长这样；
* 公式应该如何理解和记忆；
* 什么条件下可以用；
* 什么条件下不能用；
* 做题时如何识别该用它；
* 常见错误是什么。

不能只罗列公式，必须用人话解释。

## 电路类内容的特殊要求

如果处理模拟电路、电路分析、电子技术等章节，必须特别说明：

* 每个电阻的作用；
* 每个电容的作用；
* 电流源、电压源的作用；
* 晶体管、二极管、MOS 管等器件在电路中的控制关系；
* 直流分析时哪些元件等效为什么；
* 交流小信号分析时哪些元件等效为什么；
* 为什么可以建立该等效模型；
* 输入电阻、输出电阻、电压增益、电流增益如何理解；
* 公式不是只给结论，还要解释如何记忆和使用。

例如讲 BJT 时，必须说明：

* BJT 的本质控制性质是什么；
* 电流关系的实际意义是什么；
* 为什么可以使用小信号模型；
* 小信号模型中每个等效元件代表什么；
* 哪些公式考试常用，哪些公式可以剔除；
* BJT 题目的基本解题流程是什么；
* 给出简洁但覆盖关键考点的例题。

例如讲共源极放大电路时，必须说明：

* 电路中各个电阻分别有什么作用；
* 电流源为什么要这样放；
* 偏置电路的目的是什么；
* 小信号等效电路为什么这样画；
* 输入电阻、输出电阻、电压增益公式如何理解；
* 如何从电路结构判断公式形式；
* 给出简洁但覆盖关键考点的例题。

## 图像处理要求

如果教材中出现对理解或做题有帮助的图，包括但不限于：

* 电路图
* 大学物理图
* 受力图
* 光路图
* 波形图
* 曲线图
* 结构图
* 模型图
* 表格截图

必须执行以下操作：

1. 从 PDF 中截取原图。

2. 保存到 `output/assets/chXX/`。

3. 文件名使用清晰格式，例如：

   * `figure_03_01_bjt_model.png`
   * `figure_05_02_common_source.png`

```md
![BJT 小信号模型](../assets/ch03/figure_03_01_bjt_model.png)
```

如果是微积分等没有必要图像的课程，可以不生成图片文件夹，但必须在自检中说明本章无必要图片。

## 例题选择要求

教程中的例题必须简洁但全面。

例题来源优先级为：

1. 往年卷中能代表高频考点的题；
2. 课后习题中能覆盖核心方法的题；
3. 教材例题中最典型的题；
4. 必要时自行构造非常短的小例题。

例题不能堆砌过多，但必须覆盖本章主要题型。

每个例题都必须包括：

* 题目；
* 答案；
* 详细解题思路；
* 为什么这样做；
* 这道题对应哪个知识点；
* 容易错在哪里。

---

## 逐章教程完成后的自检要求

每写完一章教程后，必须自检 Markdown 格式是否正确。

自检内容包括：

* 标题层级是否正确；
* LaTeX 公式是否能正常渲染；
* 图片链接是否存在；
* 相对路径是否正确；
* 列表、表格、代码块是否闭合；
* 是否包含本规则要求的所有章节；
* 是否有明显重复或断裂内容；
* 是否有未解释的公式；
* 是否有只抄结论但没有解释的知识点。

每章必须生成自检文件：

`output/audit/chXX_md_check.txt`

---

## 审计文件要求

每章必须生成覆盖审计文件：

`output/audit/chXX_coverage.csv`

格式如下：

```csv
章节,教材页码,教材小节,处理方式,处理理由,证据,风险
```

其中“处理方式”只能使用：

* 保留
* 压缩保留
* 剔除

“处理理由”必须说明为什么这样处理。

“证据”必须写明依据，例如：

* 用户明确要求剔除；
* 往年卷未出现；
* 课后习题未覆盖；
* 高频出现在 2021 年第 3 题、2022 年第 2 题；
* 是某高频题型的前置知识；
* 教材例题中反复使用。

“风险”填写：

* 低
* 中
* 高

如果某内容被剔除但可能存在误删风险，必须标为“中”或“高”。

## 往年卷总结规则

总结往年卷时，必须完成：

1. 识别每道题属于哪一章；
2. 识别每道题属于哪个知识点；
3. 识别每道题考查什么公式、概念或方法；
4. 归纳高频题型；
5. 把同类题放在一起讲；
6. 每类题先讲知识点，再讲题；
7. 不按年份机械罗列，除非年份顺序有助于说明题型变化。

必须生成或更新：

`output/audit/past_paper_topic_map.csv`

格式如下：

```csv
年份,题号,所属章节,知识点,题型,使用公式或方法,是否高频,备注
```

## Markdown 数学公式格式强制规则

生成 Markdown 时，必须严格遵守以下数学公式格式规则。

本项目的 VS Code Markdown 预览使用 `$...$` 和 `$$...$$` 渲染数学公式。

因此：

* 行内公式使用 `$...$`
* 独立公式使用 `$$...$$`
* 禁止使用 `\(...\)` 和 `\[...\]`

---

### 1. 行内公式规则

行内公式使用 `$...$`。

正确写法：

* `令 $f_x=0$。`
* `因为 $e^y>0$，所以 $y=\cos x-1$。`
* `点 $P_1=(2k\pi,0)$ 是极大值点。`

错误写法：

* `令 \(f_x=0\)。`
* `因为 \(e^y>0\)，所以 \(y=\cos x-1\)。`
* `点 \(P_1=(2k\pi,0)\) 是极大值点。`

---

### 2. 独立公式规则

独立公式使用 `$$...$$`。

独立公式的开始符号 `$$` 必须单独占一行。

独立公式的结束符号 `$$` 必须单独占一行。

正确写法：

`$$`

`f_x=-(1+e^y)\sin x`

`$$`

错误写法：

`$$ f_x=-(1+e^y)\sin x $$`

错误写法：

`因此 $$ f_x=0 $$ 所以...`

---

### 3. 独立公式前后必须空行

独立公式前后必须各有一个空行。

正确写法：

先计算一阶偏导。

`$$`

`f_x=-(1+e^y)\sin x`

`$$`

因此令 $f_x=0$。

错误写法：

先计算一阶偏导。
`$$`
`f_x=-(1+e^y)\sin x`
`$$`
因此令 $f_x=0$。

---

### 4. 不要连续堆多个独立公式块

如果几个公式属于同一步推导，优先合并到一个 `aligned` 环境中。

正确写法：

`$$`

`\begin{aligned}`

`f_x &= -(1+e^y)\sin x \\`

`f_y &= e^y(\cos x-1-y)`

`\end{aligned}`

`$$`

错误写法：

`$$`

`f_x=-(1+e^y)\sin x`

`$$`

`$$`

`f_y=e^y(\cos x-1-y)`

`$$`

---

### 5. 有序列表中的公式规则

在有序列表或无序列表中插入公式时，必须保证公式属于当前列表项。

正确写法：

1. 先计算一阶偏导。

   `$$`

   `\begin{aligned}`

   `f_x &= -(1+e^y)\sin x \\`

   `f_y &= e^y(\cos x-1-y)`

   `\end{aligned}`

   `$$`

   因为 $1+e^y>0$ 且 $e^y>0$，所以后续只需要讨论 $\sin x$ 和 $\cos x-1-y$。

2. 再令一阶偏导为零。

错误写法：

1. 先计算一阶偏导。

`$$`

`f_x=-(1+e^y)\sin x`

`$$`

因为 $1+e^y>0$，所以...

2. 再令一阶偏导为零。

---

### 6. 禁止中文进入公式环境

中文解释文字不能写进 `$$...$$` 里面。

正确写法：

因为 $1+e^y>0$，所以只需令

`$$`

`\sin x=0`

`$$`

错误写法：

`$$`

`因为 1+e^y>0，所以 \sin x=0`

`$$`

---

### 7. 不要在公式块内部再使用 `$`

公式块 `$$...$$` 内部禁止再出现 `$`。

正确写法：

`$$`

`f(2k\pi,0)=2`

`$$`

错误写法：

`$$`

`$f(2k\pi,0)=2$`

`$$`

如果公式块内部出现 `$`，KaTeX 很容易报错：

* `KaTeX parse error`
* `Can't use function '$' in math mode`

---

### 8. 常用 LaTeX 写法

数学函数必须使用标准 LaTeX 写法。

正确写法：

* `\sin x`
* `\cos x`
* `\ln x`
* `e^x`
* `x^2+y^2`
* `f_x`
* `f_y`
* `f_{xx}`
* `f_{xy}`
* `f_{yy}`
* `\frac{a}{b}`
* `\sqrt{x}`
* `\pi`
* `+\infty`
* `\to`

错误写法：

* `sin x`
* `cos x`
* `ln x`
* `f_xx`
* `f_xy`
* `+infty`

---

### 9. 极限、偏导、积分等常见格式

极限写法：

`$$`

`\lim_{x\to+\infty,\ y\to+\infty}\left(\frac{5xy}{3(x^2+y^2)}\right)^{x^2+y^2}`

`$$`

偏导写法：

`$$`

`\begin{aligned}`

`f_x &= -(1+e^y)\sin x \\`

`f_y &= e^y(\cos x-1-y)`

`\end{aligned}`

`$$`

二阶偏导写法：

`$$`

`\begin{aligned}`

`f_{xx} &= -(1+e^y)\cos x \\`

`f_{xy} &= -e^y\sin x \\`

`f_{yy} &= e^y(\cos x-y-2)`

`\end{aligned}`

`$$`

积分写法：

`$$`

`\int_a^b f(x)\,dx`

`$$`

---

### 10. 每章完成后的检查要求

每生成一章 Markdown 后，必须检查 VS Code Markdown 预览中是否存在数学公式渲染错误。

如果预览中出现以下错误，必须修复后才能认为本章完成：

* `KaTeX parse error`
* `Can't use function '$' in math mode`
* `Expected EOF`
* `Undefined control sequence`
* `Missing }`

---

### 11. 修复原则

如果出现数学公式渲染错误，必须优先执行以下修复：

1. 检查是否把 `\(...\)` 或 `\[...\]` 用在了正文中；如果有，改为 `$...$` 或 `$$...$$`。
2. 检查所有 `$$` 是否成对出现。
3. 检查 `$$` 是否单独占一行。
4. 检查独立公式前后是否有空行。
5. 检查公式块 `$$...$$` 内部是否又出现了 `$`；如果有，删除内部 `$`。
6. 检查中文解释是否误写进了公式块。
7. 检查列表中的公式是否正确缩进。
8. 修复后重新打开 Markdown 预览，直到没有 KaTeX 报错。

---

### 12. 禁止输出的格式

禁止输出这种格式：

`\[`

`f_x=0`

`\]`

禁止输出这种格式：

`\(`

`f_x=0`

`\)`

禁止输出这种格式：

`令 \(f_x=0\)。`

必须改成：

`令 $f_x=0$。`

禁止输出这种格式：

`\[ f_x=0 \]`

必须改成：

`$$`

`f_x=0`

`$$`

## 图片截取强制规则

从 PDF 提取教材图像时，禁止使用整页截图冒充图像。

每个图片文件必须满足：

1. 图片主体必须是教材中的目标图、表、曲线、电路图、结构图或模型图。
2. 允许保留图号和图题，例如“图 6.1”。
3. 除图号、图题和必要标注外，不得包含大段正文、页眉、页脚、页码或整页留白。
4. 图片裁剪边界应贴近目标图，四周只保留少量边距。
5. 如果目标图位于页面中部，必须裁剪页面中部图像区域，不能直接保存整页。
6. 如果自动裁剪失败，必须在审计文件中标记为“图片需人工复核”，不得宣称该图已合格。
7. 如果图片明显是整页截图，必须判定为不合格并重新裁剪。

---

## 图片质量自检规则

每张保存到 `output/assets/chXX/` 的图片必须通过以下检查：

1. 图片宽度不得小于 400 px，除非原图本身很小。
2. 图片中目标图像应占图片面积的 50% 以上。
3. 图片不得包含超过 30% 的无关正文区域。
4. 图片不得包含完整页面标题、整页段落、页码和大面积空白。
5. 若图片是整页截图，必须判定为不合格并重新裁剪。
6. Markdown 中引用图片前，必须打开图片文件检查视觉效果。
7. 自检文件 `output/audit/chXX_md_check.txt` 中必须列出每张图片的检查结果。

---

## 图片失败处理规则

如果无法可靠定位目标图像边界，不得用整页截图替代。

必须执行以下之一：

1. 使用更精细的 PDF 图像提取或裁剪方法重新提取。
2. 将该图片列入 `output/audit/chXX_image_review.csv`，标记为“需人工裁剪”。
3. 在 Markdown 中暂不插入该图，并在完成报告中说明原因。

禁止在图片明显不合格的情况下宣布章节完成。

---

## 图片处理工作流

处理每章图片时，必须按以下顺序执行：

1. 先定位教材页中所有候选图像。
2. 对每个候选图像记录：

   * 图号；
   * 图题；
   * 所在 PDF 页码；
   * 对应知识点；
   * 是否必须保留。
3. 对必须保留的图像进行局部裁剪。
4. 保存到 `output/assets/chXX/`。
5. 打开保存后的 PNG 检查是否为局部图像，而不是整页截图。
6. 检查 Markdown 图片链接是否正确。
7. 在审计文件中记录图片处理结果。

---

## 图片审计文件要求

每章必须生成图片审计文件：

`output/audit/chXX_image_audit.csv`

格式如下：

```csv
章节,图片文件名,教材页码,图号或表号,对应知识点,处理方式,是否局部裁剪,是否含无关正文,是否清晰,是否通过检查,备注
```

其中：

* `是否局部裁剪` 只能填写：是 / 否
* `是否含无关正文` 只能填写：是 / 否
* `是否清晰` 只能填写：是 / 否
* `是否通过检查` 只能填写：是 / 否

如果 `是否局部裁剪=否` 或 `是否含无关正文=是`，则 `是否通过检查` 必须为“否”。

如果某张图片没有通过检查，不得在教程 Markdown 中作为正式图片引用，除非在完成报告中明确说明“该图需人工复核”。

## 文本质量强制验收规则

每章教程不得只满足模板结构，必须通过以下文本质量检查：

1. “本章常见考法”中列出的每一种考法，必须在“典型题型总结”中至少有一个对应题型。
2. 每个“必须掌握”考点，必须至少关联以下证据之一：

   * 往年卷题号；
   * 课后习题题号；
   * 教材典型例题；
   * 用户明确要求。
3. 每个核心知识点必须包含：

   * 概念本质；
   * 为什么引入；
   * 公式每一项含义；
   * 做题识别信号；
   * 使用条件；
   * 常见误区；
   * 至少一个例题或反例。
4. 如果某章前面列出多类常见考法，后面只展开 1-2 类，判定为不合格。
5. 典型题型总结不得少于本章高频考法数量的 80%。
6. 每个例题必须说明：

   * 来源；
   * 对应知识点；
   * 第一步为什么这样做；
   * 中间关键步骤为什么成立；
   * 答案如何检查；
   * 常见错误。
7. 每章完成后，必须生成 `output/audit/chXX_text_quality_check.txt`。
8. 如果文本质量检查未通过，不得宣布章节完成。

## 章节完成门禁规则

每章完成前必须同时满足以下条件：

1. 已生成教程 Markdown：`output/tutorials/chXX.md`
2. 已生成习题课 Markdown：`output/exercise_classes/chXX.md`
3. 已生成覆盖审计文件：`output/audit/chXX_coverage.csv`
4. 已生成 Markdown 自检文件：`output/audit/chXX_md_check.txt`
5. 已生成文本质量自检文件：`output/audit/chXX_text_quality_check.txt`
6. 若本章使用图片，已生成图片审计文件：`output/audit/chXX_image_audit.csv`
7. 所有 Markdown 图片链接均有效。
8. 所有图片均为局部裁剪图，不得是整页截图。
9. “本章常见考法”与“典型题型总结”覆盖关系通过检查。
10. 所有必须掌握考点均有证据来源。
11. 所有核心公式均有解释，不得只罗列公式。
12. 所有例题均有详细讲解，不得只给答案。

如果以上任一项未完成，不得输出“本章完成报告”，只能输出“本章未完成问题清单”。

## 高质量章节判定标准

一个章节只有同时满足以下标准，才可以称为“完成”：

1. 教程不是简单公式汇总，而是能解释学生为什么要这样理解。
2. 习题课不是答案堆砌，而是能讲清楚题型识别和做题思路。
3. 图片不是整页截图，而是围绕知识点的局部清晰截图。
4. 审计文件能说明保留、压缩保留、剔除的依据。
5. 自检文件能发现并记录 Markdown、公式、图片和文本质量问题。
6. 完成报告能真实反映未解决风险，不得为了完成而隐瞒问题。
