# 第一性原理思考 — First Principles Thinking

> 这七个字，做的事情就是强制打断 AI 的类比推理，逼它回到问题的本质去思考，不要参考别人的方案，从最基本的事实出发，重新推导。

## 核心原理

现在的 AI，很多都还是在做类比推理——你跟它说"写一个过滤函数"，它会在训练数据里找到几万个类似的过滤函数，然后给你写一个看起来差不多的出来。这个过程很快，结果也能用，但它跳过了最关键的步骤：**从本质出发思考**。

第一性原理的核心就一句话：**剥离所有假设，回到最基本的事实和约束条件，从零开始推导。**

当你要求 AI "从第一性原理出发"时，它会：
1. 识别并剥离所有未经证实的假设
2. 回到问题的最基本事实和物理/逻辑约束
3. 不参考"别人怎么做"，从本质重新推导方案
4. 质疑每一个"理所当然"的设计选择

## 你的任务

当用户在任何请求后附加"从第一性原理出发"（或触发本 skill）时，你必须切换思维模式：

### 第一步：剥离假设
- 列出当前问题中被视为"理所当然"的所有假设
- 对每个假设问：这个假设真的成立吗？为什么？
- 区分"事实"和"观点/惯例"

### 第二步：回归本质
- 明确这个问题的**最根本目标**是什么
- 识别不可改变的**物理/逻辑约束**（不是人为约束）
- 从零开始：如果不参考任何已有方案，这个问题该怎么解？

### 第三步：重新推导
- 从最基本的事实和约束出发，一步步推导
- 每一步都要问：为什么必须是这一步？有没有替代路径？
- 不要被"行业惯例"或"主流做法"绑架

### 第四步：对比验证
- 将推导出的方案与原始方案对比
- 如果不同：原始方案中的哪些假设导致了偏差？
- 明确回答：从第一性原理来看，正确的做法是什么？

## 输出格式

当被触发时，你的回答应该包含：

```
## 🔍 假设剥离
[列出所有被剥离的假设，标记哪些成立、哪些不成立]

## 🧱 本质回归
[问题的最根本目标 + 不可改变的约束条件]

## 📐 从零推导
[从约束条件出发，一步步推导出方案]

## ⚖️ 对比分析
[新方案 vs 原方案的差异，以及差异的根本原因]
```

## 应用场景

这个 Prompt 与具体领域无关，适用于：
- **修 BUG**：不被表面症状迷惑，找到最本质的原因
- **设计架构**：不被现有框架绑架，从需求本质推导架构
- **写代码**：不复制粘贴类似实现，从功能本质写最优解
- **商业决策**：剥离行业惯例，从商业本质推导策略
- **写文章**：不被流行观点带偏，从论题本质构建论证
- **个人决策**：不被社会期待干扰，从自己真正想要什么出发

## 使用方式

无需特殊命令。在任何请求后面加上"从第一性原理出发"即可：

```
帮我修复这个内存泄漏，从第一性原理出发。
```

```
设计这个 API 的权限系统，从第一性原理出发。
```

```
我要不要换工作？从第一性原理想清楚。
```
