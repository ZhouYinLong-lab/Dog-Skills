# MkDocs Material 排版强制规范

> 来源: SmartTutor Generator v1.1 (莫弈, 2026)
> 基于 MkDocs Material 实际渲染效果总结的强制性规范

---

## 规则 1: 空行隔离原则 (最高优先级)

### 1.1 列表隔离
在任何无序列表(`-`)或有序列表(`1.`)的**首行之前**,必须强制插入一个物理空行。

```
✅ 正确:
这是正文内容。

- 列表项1
- 列表项2

❌ 错误(粘连):
这是正文内容。
- 列表项1
- 列表项2
```

### 1.2 区块间隙
代码块(` ``` `)、引用块(`>`)、Admonition 弹窗(`!!!`)前后均必须各保留一个物理空行。

```
✅ 正确:
这是正文。

```python
print("hello")
```

继续正文。

❌ 错误:
这是正文。
```python
print("hello")
```
继续正文。
```

### 1.3 严禁粘连
绝对禁止列表符号紧跟在正文末尾。

```
❌ 最高级别格式错误:
...标签。- 列表项
...正文。1. 有序列表

✅ 正确:
...标签。

- 列表项
```

## 规则 2: 粗体空格隔离 (强制)

### 2.1 基本规则
所有使用 `**` 标记的粗体文本,必须在 `**` 标记前后各添加一个空格。

```
✅ 正确: 这是 **粗体文字** 示例
❌ 错误: 这是**粗体文字**示例
```

### 2.2 中文标点隔离
当 `**` 紧邻中文标点(。，：；！？（）【】《》)时,同样需要空格隔离。

```
✅ 正确: 核心是 **PDCA 循环** ：
❌ 错误: 核心是 **PDCA 循环**：
```

### 2.3 括号隔离
当 `**` 紧邻中文或英文括号时,需要空格隔离。

```
✅ 正确: **公钥** （公开）
❌ 错误: **公钥**（公开）
```

### 2.4 表格内适用
表格单元格内的粗体文本同样遵守此规则。

```
✅ 正确: | 加密速度 | **快** （适合大量数据） |
❌ 错误: | 加密速度 | **快**（适合大量数据） |
```

### 2.5 列表项适用
列表项中的粗体文本同样遵守此规则。

```
✅ 正确: - **默认拒绝** ：没有明确允许的流量一律拒绝
❌ 错误: - **默认拒绝**：没有明确允许的流量一律拒绝
```

### 2.6 验证方法
使用正则搜索违规项: `[^\s]\*\*[^\s\*]`
(匹配 `**` 前后缺少空格的情况)

## 规则 3: MkDocs Admonition 使用

| 类型 | 语法 | 用途 |
|------|------|------|
| 信息 | `!!! info "标题"` | 术语解释、背景知识 |
| 示例 | `!!! example "标题"` | 生活化案例、场景演示 |
| 提示 | `!!! tip "标题"` | 经验技巧、最佳实践 |
| 注意 | `!!! note "标题"` | 补充说明 |
| 警告 | `!!! warning "标题"` | 常见错误、注意事项 |
| 危险 | `!!! danger "标题"` | 安全风险、数据丢失风险 |

```
✅ 正确(含空行):

这是正文。

!!! info "什么是API?"
    API就像餐厅的服务员——你点菜(发请求),
    服务员传给厨房(服务器),再把菜端回来(响应)。

继续正文。
```

## 规则 4: 标题层级

```markdown
# 一级标题 — 带趣味副标题
## 1. 二级标题 — 带数字标号
### 三级标题 — 无编号
```

- 一级标题: 每章开头,可带趣味副标题
- 二级标题: 带数字标号(1. 2. 3.)
- 三级标题: 无编号
- 不跳级(一级→三级 不允许)

---

## 规则 5: 数学公式 (MathJax)

```
行内: $E = mc^2$
块级:
$$
\frac{d}{dx}e^x = e^x
$$
```

- 行内公式用单 `$`
- 块级公式用双 `$$`
- 公式前后保留空行(块级)

---

## 规则 6: 代码块

```markdown
这是正文。

```python
# 代码必须含中文注释
def hello():
    print("Hello World")  # 输出问候语
```

继续正文。
```

- 准确语言高亮(python/javascript/bash/yaml)
- 命令输出必须展示预期结果
- 代码块前后各一个空行

---

## 规则 7: 多系统 Tabbed Content

```markdown
=== "Windows"
    在 Windows 上的操作步骤...

=== "macOS"
    在 macOS 上的操作步骤...

=== "Linux"
    在 Linux 上的操作步骤...
```

- 前后各保留一个空行
- 按字母顺序排列系统
