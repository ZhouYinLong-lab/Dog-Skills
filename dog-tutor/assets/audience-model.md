# 四级受众模型

> 来源: SmartTutor Generator v1.1 (莫弈, 2026)

---

## 受众级别定义

| 级别 | 标识符 | 描述 | 比喻密度 | 步骤粒度 | 前置假设 |
|------|--------|------|:---:|:---:|------|
| 零基础 | `absolute_beginner` | 完全零基础,需要生活化比喻和极细步骤 | high | micro | none |
| 初学者 | `beginner` | 有基础概念,需要清晰引导和适度比喻 | medium | standard | basic |
| 进阶者 | `intermediate` | 有实践经验,需要深度和最佳实践 | low | condensed | solid_foundation |
| 专家 | `expert` | 资深从业者,需要前沿技术和架构视角 | minimal | overview | expert_level |

---

## 自动识别规则

| 标题特征 | → 级别 |
|----------|--------|
| 含"零基础""新手""入门""小白""第一次" | absolute_beginner |
| 含"进阶""实战""精通""深入""高级" | intermediate |
| 含"架构""原理""源码""内核""底层" | expert |
| 无特殊标记(默认) | beginner |

### 动态调整机制

- 每章生成后询问: "这个难度合适吗?"
- 用户反馈"太简单" → 提升一级
- 用户反馈"太难" → 降低一级
- 特定章节可单独调整

## 各级别写作策略

### absolute_beginner (零基础)

```
写作策略:
- 每个操作配截图描述
- 每步都要验证
- 避免任何专业术语(或用比喻解释)
- 使用大量"就像..."句式
- 步骤编号粒度: 每个点击动作一个步骤
- 失败案例 > 成功案例(降低焦虑)

示例:
"打开终端就像打开家里的电表箱——看起来很复杂,
但其实我们只需要操作几个简单的开关。"
```

### beginner (初学者)

```
写作策略:
- 关键步骤详细,次要步骤简化
- 术语首次出现用 !!! info 解释
- 每个概念配1个比喻
- 步骤编号粒度: 每个操作目标一个步骤
- 提供常见错误及解决方案

示例:
"cd 命令就像在文件夹之间'跳跃'。
cd /home 就是跳到 home 这个文件夹里。"
```

### intermediate (进阶者)

```
写作策略:
- 侧重原理和最佳实践
- 提供对比分析(方案A vs 方案B)
- 比喻用于解释抽象概念,不对基础概念使用
- 代码示例展示多种实现方式
- 强调性能/安全/可维护性

示例:
"管道操作符 | 实现了进程间通信的零拷贝数据传输,
相比临时文件方案,I/O开销降低约40%。"
```

### expert (专家)

```
写作策略:
- 提供架构视角和前沿技术
- 讨论设计权衡和适用边界
- 引用学术论文/源码实现
- 比喻仅用于解释全新概念
- 提供可扩展的参考实现

示例:
"Linux Cgroup v2 的层级结构设计借用了
文件系统的单一继承模型,但通过 threaded 模式
解决了 v1 中线程级控制的粒度问题。"
```
